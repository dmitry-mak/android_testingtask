### BIN-reader

**Тестовое задание для прохождения практики по специальности "андройд-разработчик"**

#### Задание:

- Реализовать получение с сайта https://binlist.net/  данных о банке по вводимому bin-номеру:
Пользователь вводит BIN банковской карты и видит доступную информацию о нём (страна, координаты, тип карты(visa и тп), данные банка(url, телефон, сайт, город)) загруженную с https://binlist.net/ 
Опционально* 
Нажатие на URL банка, телефон банка или координаты страны отправляет пользователя в приложение, которое может обработать эти данные (браузер, звонилка, карты)  
Пользователь имеет возможность перейти на экран со списком карточек истории запросов

- Реализовать запись истории предыдущих запросов, в виде карточек с данными, которые были получены с ендпоинта:
Список карточек истории запросов BIN номеров, с данными которые ранее были полученны с ендпоинта.  
История предыдущих запросов выводится списком. 
История предыдущих запросов не теряется при перезапуске приложения. 

Необходимый стэк:
- Kotlin, 
- MVVM, 
- Coroutineб 
- обязательно использовать DI (koin, dagger, hilt), 
- предпочтительно использовать Jetpack Compose, 
- придерживаться Clean Architecture.

#### Выполненные требования:
- разделение на 2 экрана:
  1. Экран для поиска по BIN номеру:
  - ввод BIN-номера (номер ограничен 6-8 цифрами)
  - получение информации от API https://binlist.net/
  - отображение доступной информации: тип карты (scheme), категория карты (type), бренд карты (brand), страна выдачи (country), координаты (coordinates), банк (bank)- название, номер телефона, адрес электронной почты
  2. Экран отображения истории запросов:
  - сохранение всех запросов в локальную базу данных (Room)
  - отображение истории в виде списка карточек
  - сохранение данных после перезапуска приложения
  - очистка истории поиска (дополнительно, вне технического задания)

- проект разделен на три слоя:
  - слой данных (data), включающий 3 пакета: local, remote, repository
  - слой с бизнес-логикой (domain), включающий 3 пакета: model, repository, usecase
  - слой с UI (uiscreens), включающий 3 пакета: history, search, navigation

- При реализации проекта использованы:
  - язык - Kotlin
  - UI- JetpackCompose
  - взаимодействие с сетью- Retrofit
  - управление базой данных- Room
  - управление зависимостями (DI)- Koin
